# AI ガードレール — お絵かきYouTuberアカデミー HP

| 項目 | 値 |
|---|---|
| Spec Slug | `oekaki-youtuber-academy-hp` |
| 目的 | AI（Claude Code 等）に本サイトの編集を任せる際に、踏んではいけない地雷と守るべき価値観を明示する |
| 適用対象 | 本サイトの編集・運用作業を行うすべての AI エージェント |
| 最終更新 | 2026-06-29 |

---

## 0. 大原則

> **「印象向上と信頼構築」が第一目的。CV 数最大化ではない。** 売り込み色を強める変更は原則として却下し、必要なら人間に確認する。

---

## 1. コンテンツ・表現ガイドライン

### 1.1 やってはいけない（コピー）

| カテゴリ | NG 例 | 理由 |
|---|---|---|
| 絶対断定 | 「誰でも必ず月収 100 万円」「100% 稼げる」 | 景品表示法・特商法上のリスク（NFR-C2） |
| 締切煽り | 「あと 24 時間で締切」「残り 3 名のみ」 | ブランディング目的に反する（広告 LP の役割） |
| 誇張記号の多用 | 「！！！」「★★★★★」 | 本サイトの落ち着いたトーンを破壊 |
| ネガティブ煽り | 「このままだと一生稼げません」 | 同上 |
| 比較貶め | 「他のスクールは詐欺」 | 信頼を損ねる |
| 数値の根拠不明な改ざん | 月収を勝手に水増し | 元動画コンテンツとの不整合（NFR-C3） |

### 1.2 やってよい（コピー）

| カテゴリ | 例 |
|---|---|
| 控えめな実績提示 | 「月収 43 万円を達成された方もいらっしゃいます」 |
| 個人差注記 | 「※ 結果には個人差があります」 |
| 共感的な問いかけ | 「『好きなことで生きていきたい』と願う方へ」 |
| 物語で語る | 「東京で美容師として働いていた頃…」 |

### 1.3 表現トーン

- **語尾**: 「です」「ます」基調。命令形（「〇〇しろ」「〇〇しなさい」）は使わない
- **句読点**: ヒーロー・哲学・LINE CTA など強調セクションでは**句読点を省略**する（ブランディング設計）
- **絵文字**: 原則使わない。例外は `●` のような装飾的記号のみ
- **太字 (`<strong>`)**: ストーリー内の核心 1〜2 語のみ

## 2. 構造・コードガイドライン

### 2.1 触ってはいけない

| 対象 | 理由 |
|---|---|
| 既存セクションの順序（Hero〜Footer） | 視線設計と心理設計の根幹（requirements §4） |
| `vercel.json` の rewrites | URL ルーティング全体に影響 |
| 受講生 6 名の YouTube 動画 ID | 講師の許諾・正典との対応（requirements §4.3） |
| Hero の 4 本の黄色斜めストライプ装飾 | ブランド統一性 |
| `images/portrait-transparent.png`（背景透過写真） | 講師の同意プロセスを経た正本 |

### 2.2 触る前に確認するもの

| 対象 | 確認手段 |
|---|---|
| プロフィール 6 facts | requirements.md §4.6 FR-502 |
| Curriculum 6 項目 | requirements.md §4.7 FR-602 |
| FAQ 5 項目 | requirements.md §4.5 FR-401 |
| お問い合わせの mailto subject | requirements.md §4.8 FR-701 |

### 2.3 推奨パターン

| 変更タイプ | 良い実践 |
|---|---|
| テキスト変更 | 該当ファイルを Read してから Edit |
| 画像差し替え | 古い画像は履歴に残るので削除せずに上書き |
| 新セクション追加 | 必ず requirements / design / tasks に反映 |
| CSS 変更 | デザイントークン（`:root` の CSS 変数）を経由 |

## 3. プライバシー・倫理

### 3.1 個人情報の取扱

| 種別 | 扱い |
|---|---|
| 講師の本名・顔写真 | ✅ 公開（A-1 で合意済） |
| 受講生の本名 | ❌ 公開しない（「○○さん」止まり） |
| 受講生の住所・電話 | ❌ 絶対公開しない |
| 講師の家族（子供）の名前・顔 | ❌ 公開しない |
| 講師の自宅・アトリエの位置情報 | ❌ 公開しない |

### 3.2 第三者著作物

| 種別 | 扱い |
|---|---|
| YouTube サムネ画像 | `img.youtube.com` から API 経由で参照（埋め込み相当） |
| YouTube 動画自体 | 埋め込まずリンクのみ（帯域・許諾範囲） |
| メディア掲載のスクリーンショット | 出典明記の上で使う場合は事前に法務確認 |

### 3.3 差別・ハラスメント表現

- 性別・年齢・職業・地域などに対する差別的表現は全面禁止
- 「専業主婦」「派遣社員」「飲食店勤務」は事実描写であり差別ではないが、貶める文脈で使わない
- 「旦那を見返す」は元コンテンツの言葉だが、本サイトでは「家族との時間を守った」など中立的に言い換える

## 4. 運用ルール

### 4.1 push 前の必須チェック

- [ ] HTML が正しく整形されている（タグの閉じ忘れなし）
- [ ] 画像パスが正しい
- [ ] 受講生 6 名の名前・属性・月収が requirements §4.3 と一致
- [ ] 「※ 結果には個人差があります」が Voice セクションに残っている

### 4.2 push しないほうがいいシグナル

- 大量の変更を一度に行うとき（10 ファイル以上、500 行以上）
- 既存セクションを丸ごと削除するとき
- 受講生・講師の情報を改変するとき
- `vercel.json` を変更するとき

これらが必要な場合は、まず人間（colabpaint）に確認をとる。

### 4.3 commit メッセージ

- 1 行目: 何を変えたか（Imperative, 50 文字以内）
- 空行
- 2 段目以降: なぜ・どこを（必要なら）

例:
```
Update hero copy to remove unnecessary punctuation

- Remove "、" from "ポスカを使って、楽しく描くことで"
- Aligns with brand voice guideline (no punctuation in hero)
```

## 5. 緊急時の判断基準

| 状況 | 判断 |
|---|---|
| 受講生・講師から削除依頼が来た | **最優先で即実行**。文書を確認しなくても先に非表示化 |
| 法的指摘が入った（景品表示法 / 著作権 等） | 当該部分を非表示化、運営者で内容確認、24h 以内に対応 |
| 第三者から「貴サイトのこの主張は誤りでは」と指摘された | 事実確認 → 元動画と照合 → 必要なら訂正 |
| サイトが意図せず変更された（外部侵入の可能性） | runbook §1.1 の手順、最後の正常な commit に revert、2FA 再確認 |

## 6. AI が「これは大丈夫？」と迷ったときのフロー

```
1. このドキュメント (§1〜5) で答えがあるか確認
2. requirements.md / design.md / threat-model.md を確認
3. それでも判断できなければ、人間に質問する
   "[何を] [なぜ] 必要なのかが不明なので、確認させてください。
    [想定されるリスク: A, B] [代替案: X, Y]"
4. 自己判断で進めない
```

## 7. AI が「嘘をつかない」ためのルール

CLAUDE.md（グローバルルール）の遵守を再掲:

- 根拠のないことは言わない
- 「質が上がった」「良くなった」等の評価は、比較基準と根拠を示せる場合のみ発言
- 推測する場合は「推測ですが」と明示する
- できないことは「わかりません」「できません」と正直に言う
- ファイルを読まずに「読みました」と言わない
- push していないのに「公開しました」と言わない

このサイトの編集時も同様に適用する。
