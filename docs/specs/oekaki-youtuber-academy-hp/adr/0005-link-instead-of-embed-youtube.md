# ADR-0005: YouTube 動画は埋め込まずリンクのみとする

| 項目 | 値 |
|---|---|
| 状態 | Accepted |
| 日付 | 2026-06-28 |
| 関連要件 | FR-204, NFR-P3, C-5 |

---

## 1. 文脈・問題

Voice セクションでは 6 人の受講生対談動画（YouTube）を紹介する。動画を「埋め込み」にするか「リンク」にするかを決める必要がある。

## 2. 検討した選択肢

| 案 | 概要 | 利点 | 欠点 |
|---|---|---|---|
| A. iframe 埋め込み | YouTube の埋め込みプレイヤー | サイト内で完結、視聴体験良好 | (a) ページ重くなる（iframe 6 個分の JS）、(b) Cookie 同意ロジックが必要、(c) Core Web Vitals 悪化、(d) GDPR/プライバシー配慮 |
| B. **サムネ表示 + リンクで YouTube 遷移** | カードでサムネ表示、クリックで YouTube へ | 軽い、Cookie ゼロ、UX 簡素 | サイト離脱が発生 |
| C. lite-youtube-embed（軽量カスタム要素） | 初回表示は軽く、クリック後に iframe ロード | 中間策 | カスタム要素の保守、依然 Cookie 配慮必要 |

## 3. 決定

**案 B: サムネ + リンク** を採用。

### 実装ポイント

- サムネは `https://img.youtube.com/vi/{動画ID}/maxresdefault.jpg` から取得
- `<a href="https://youtu.be/{動画ID}" target="_blank" rel="noopener">`
- カード全体がクリッカブル
- `loading="lazy"` で初期 LCP を改善
- 再生マーク（▶）を CSS で重ねて「動画」と明示

## 4. 結果

- ページ重量を最小化（NFR-P3: 2 MB 上限を維持）
- Cookie 同意フローを回避
- GDPR / 個人情報保護法上の負担なし
- YouTube 側に視聴データが残るので、講師にとっても再生数のシグナルが集約される

## 5. トレードオフ

- 訪問者がサイトから離脱する（ただし広告 LP ではないので CV 直結ではない、許容）
- YouTube 動画が削除された場合、サムネが 404 になる → tasks.md T-605 のリスクとして追跡
