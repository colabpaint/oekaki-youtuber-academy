# ADR-0006: GitHub リポジトリを Public とする

| 項目 | 値 |
|---|---|
| 状態 | Accepted |
| 日付 | 2026-06-28 |
| 関連要件 | C-3, I-1（threat-model） |

---

## 1. 文脈・問題

Vercel Hobby プランから GitHub プライベートリポジトリを連携する場合、Vercel 側で「個人利用」と判定されつつも、いくつかの機能制約や利用規約の変動リスクがある。一方、本サイトはコード自体に秘密値や独自ロジックを含まない（静的 HTML/CSS のみ）。

## 2. 検討した選択肢

| 案 | 概要 | 利点 | 欠点 |
|---|---|---|---|
| **A. Public リポジトリ** | 誰でも閲覧可能 | Vercel Hobby と相性 100%、コード公開で信用性 ↑ | コードがコピーされる可能性（ただし公開 HTML はもともとブラウザで見える） |
| B. Private リポジトリ + Vercel Hobby | クローズドソース | コードがコピーされない | Vercel Hobby の利用規約に細かな制約。商用判定のリスク |
| C. Private + Vercel Pro 有料 | 商用安全 | フル機能 | 月額発生 |

## 3. 決定

**案 A: Public リポジトリ** を採用。

### 理由

1. **コードは元々ブラウザで見える**: 静的 HTML/CSS なので "ソースを隠す" こと自体に意味が薄い
2. **秘密値ゼロ**: API キー・トークン・DB 接続情報なし
3. **Vercel Hobby と整合**: ライセンス上の懸念が最小
4. **コピー対策**: コピーされても問題ない範囲のみを置く（独自のロジック・知的財産は別管理）

## 4. 結果

- `colabpaint/oekaki-youtuber-academy` を Public で運用
- `.gitignore` で macOS の `._*` や `.env*` を除外
- 受講生の YouTube 動画 ID は公開情報なのでそのままコミット
- 講師の本名・顔写真は要件 A-1 で公開合意済み

## 5. トレードオフ

- 第三者がコードをそのまま流用してサイトを建てる可能性
  - 対策: ブランド名・写真は著作権で保護される
- フォーク数や Star 数が公開される（プライバシー上は問題なし）
- threat-model I-1（秘密値漏洩）は「そもそも秘密値を持たない」設計で対処
