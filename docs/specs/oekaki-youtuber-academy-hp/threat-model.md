# 脅威モデル (STRIDE) — お絵かきYouTuberアカデミー HP

| 項目 | 値 |
|---|---|
| Spec Slug | `oekaki-youtuber-academy-hp` |
| 採用モデル | STRIDE (Spoofing / Tampering / Repudiation / Information Disclosure / Denial of Service / Elevation of Privilege) |
| 評価対象 | 公開URL `https://oekaki-youtuber-academy.vercel.app/` および主要リポジトリ `colabpaint/oekaki-youtuber-academy` |
| 評価日 | 2026-06-29 |

---

## 0. システム特性まとめ

本プロダクトは **完全静的サイト** であり、特性として:

- バックエンドなし／DB なし／認証なし／PII 収集なし
- 第三者 JS なし（Google Fonts のみ）
- ユーザー入力なし（フォームなし）
- mailto: と外部 URL リンクのみ

→ 一般的な Web アプリより**脅威面（attack surface）は狭い**が、それでも以下の観点を点検する。

## 1. STRIDE 分析

### 1.1 Spoofing（なりすまし）

| ID | 脅威 | 影響度 | 発生可能性 | 緩和策 | 状態 |
|---|---|---|---|---|---|
| S-1 | 第三者が類似ドメイン（`oekaki-youtuber.com` 等）で偽サイトを立てて訪問者を誘導する | High | Mid | (a) 公式ドメイン確定後に SNS で告知（FUT-1）、(b) Google での名前検索結果のモニタリング | **未対応**（独自ドメイン未取得） |
| S-2 | 第三者が `colabpaint` の GitHub アカウントを乗っ取り、リポジトリを改ざんしてサイト内容を差し替える | High | Low | (a) GitHub 2FA を有効化、(b) Vercel アカウントも 2FA、(c) 不審なログイン通知を有効化 | ⚠️ **GitHub の 2FA 未確認**。Vercel の 2FA は当面スキップ状態 |
| S-3 | 講師ユウになりすました偽 LINE 公式アカウントが立てられる | High | Mid | 公式アカウントを「認証済みアカウント」に申請、サイト上で公式アカウントの ID を明記 | LINE 開設時に対応（T-501 と併せて） |

### 1.2 Tampering（改ざん）

| ID | 脅威 | 影響度 | 発生可能性 | 緩和策 | 状態 |
|---|---|---|---|---|---|
| T-1 | リポジトリ侵入による HTML/CSS 改ざん | High | Low | S-2 の対策と同等（2FA） | ⚠️ 未確認 |
| T-2 | 中間者攻撃で HTTP 配信を改ざん | High | Very Low | HTTPS 強制 + HSTS | ✅ 対応済（Vercel デフォルト） |
| T-3 | キャッシュポイズニング（CDN 層） | High | Very Low | Vercel 側で対策。本サイトはユーザー入力なしで差別化キー無し | ✅ Vercel に依存 |
| T-4 | 受講生写真や YouTube リンクの差し替え | Mid | Low | 2FA + リリース時チェックリストで目視確認 | ✅ チェックリストあり（tasks §1.1） |

### 1.3 Repudiation（否認）

| ID | 脅威 | 影響度 | 発生可能性 | 緩和策 | 状態 |
|---|---|---|---|---|---|
| R-1 | 「掲載許諾を得ていない」と受講生・第三者から主張される | High | Low | (a) 文書による許諾記録、(b) 申し立てがあったら即時非公開、(c) `git log` で変更履歴を保存 | ⚠️ 文書許諾の記録は要整備 |
| R-2 | 講師の「言った／言わない」問題で運営対応の記録が残らない | Mid | Mid | お問い合わせは mailto: 経由のためメール履歴に残る | ✅ メール履歴で対応可 |

### 1.4 Information Disclosure（情報漏洩）

| ID | 脅威 | 影響度 | 発生可能性 | 緩和策 | 状態 |
|---|---|---|---|---|---|
| I-1 | 公開リポジトリに API キー / シークレットが混入 | Critical | Low | (a) 静的サイトのため秘密値そのものがない、(b) `.gitignore` に `.env*` を追加（将来導入時） | ✅ 現状秘密値なし |
| I-2 | 講師の私生活情報（住所・電話）が誤って掲載される | High | Low | リリースチェックリストに「個人情報の掲載漏れがないか」を追加 | ⚠️ チェック項目に追加要 |
| I-3 | 受講生の本名や住所が公開される | High | Low | Voice カードは「○○さん」「元・看護師」止まり。本名・住所なし | ✅ 現状ガード済 |
| I-4 | プライバシーポリシー未掲載で「データ取扱の透明性が低い」と外部から指摘 | Mid | Mid | T-503 で本文作成 | ⚠️ 未対応（GAP-3） |
| I-5 | mailto: アドレスがスパムボットに収集される | Mid | High | (a) 簡易な obfuscation（`info [at] example [dot] com`）または、(b) Cloudflare 等の email-obfuscation、(c) 専用エイリアスで分離 | ⚠️ 現状無対策（mailto は素のまま） |

### 1.5 Denial of Service（DoS）

| ID | 脅威 | 影響度 | 発生可能性 | 緩和策 | 状態 |
|---|---|---|---|---|---|
| D-1 | 大量アクセスによる Vercel Hobby プランの帯域超過 | Mid | Low | (a) 月間 100 GB 無料、(b) 超過時の自動課金は無効、(c) 必要なら一時的に GitHub Pages へ切替 | ✅ Hobby は自動課金なし |
| D-2 | Google Fonts や YouTube img CDN の障害 | Low | Low | Google / YouTube に依存。本体表示は劣化のみ（テキストはシステムフォントで描画継続） | ✅ Graceful degradation |
| D-3 | mailto: の悪用（迷惑メール大量送信） | Low | Low | サーバー側で受信。SPAM フィルタに依存 | ✅ Gmail 等の SPAM 対策に依存 |

### 1.6 Elevation of Privilege（権限昇格）

| ID | 脅威 | 影響度 | 発生可能性 | 緩和策 | 状態 |
|---|---|---|---|---|---|
| E-1 | サイト管理者権限の不正取得（Vercel 側） | Critical | Very Low | Vercel の 2FA、信頼できる端末からのみ操作 | ⚠️ 2FA 未設定 |
| E-2 | GitHub PAT / OAuth トークンの漏洩 | High | Low | gh CLI のトークンは keyring に保存。`~/.config/gh/hosts.yml` を平文で持たない | ✅ keyring 保存確認済（`gh auth status` 出力に "keyring" 表記） |

## 2. 残存リスク（Residual Risk）

| 残存リスク | 受容判断 | 受容理由 |
|---|---|---|
| GitHub・Vercel の 2FA 未設定 | **受容しない** — 早期対応推奨 | 影響度が High。設定コスト数分なのに後回しは合理的でない |
| プライバシーポリシー未掲載 | 一時受容 | フォーム未実装で PII 収集していない。法的義務発火前 |
| OGP 未設定 | 一時受容 | SNS シェアが本格化するまで影響限定 |

## 3. 推奨対応アクション（優先度順）

| # | アクション | 紐づくリスク | 推定所要時間 |
|---|---|---|---|
| 1 | GitHub アカウントの 2FA 有効化 | S-2, T-1, E-2 | 10 分 |
| 2 | Vercel アカウントの 2FA 有効化 | E-1, T-3 | 10 分 |
| 3 | mailto: の軽い obfuscation または専用エイリアス導入 | I-5 | 30 分 |
| 4 | リリースチェックリストに「個人情報の漏れ確認」を追加 | I-2 | 5 分（このタスク内で本ドキュメントに反映） |
| 5 | プライバシーポリシー本文作成（フォーム導入前でも掲載推奨） | I-4 | 1-2 時間 |
| 6 | 受講生掲載許諾の文書記録整備 | R-1 | 講師経由で 1 週間 |
| 7 | LINE 公式アカウント認証取得 | S-3 | LINE 公式の審査次第 |
| 8 | 独自ドメイン取得とブランド告知 | S-1 | 1 時間 + 浸透待ち |

## 4. 攻撃面マップ（要約）

```
                          ┌─────────────────┐
                          │  訪問者ブラウザ   │
                          └────────┬────────┘
                                   │
                  ┌────────────────┼─────────────────┐
                  │                │                 │
                  ▼                ▼                 ▼
        ┌──────────────────┐ ┌──────────────────┐ ┌────────────────┐
        │  HTTPS (Vercel)  │ │  Google Fonts    │ │  img.youtube.com│
        │  HSTS + Brotli   │ │  (フォント取得)   │ │  (サムネ取得)    │
        └────────┬─────────┘ └──────────────────┘ └────────────────┘
                 │
       ┌─────────┴─────────┐
       │  Vercel Edge CDN  │
       └─────────┬─────────┘
                 │ webhook
       ┌─────────┴────────────────┐
       │  GitHub Repository       │ ◀── 攻撃者の主要侵入口
       │  (push 権限が侵害された   │
       │   場合の影響: 高)         │
       └──────────────────────────┘
                 │
       ┌─────────┴────────────────┐
       │  ローカル開発端末          │
       │  (gh CLI / 2FA / 物理)    │
       └──────────────────────────┘
```

**最重要保護対象**: GitHub アカウント（信頼の起点）。これが侵害されると全リスクが連鎖的に顕在化する。
